Lex and Yacc Exercises

For each of the exercises below, you should use the file names as given in the solution template.

House points for the optional questions are given once per house, to the student who submits the earliest.

Some of the solutions may be available online, even in the tutorials provided as references. But you are advised to code them from scratch (no cut-paste) for understanding. Any cut-paste will be detrimental to your learning and will affect your performance in exams for sure.

  1. Toy exercises to get warmed up
    1. count-lines.l Write a lex file to count the number of lines in the input.
    2. count-lwc.lWrite a lex file to count the number of lines, words, and characters in the input. A word is contiguous string of alphanumeric characters (alphabets and numbers only). Note that this definition is different from that used by the 'wc' program.
    3. count-num.l Write a lex file to count the number of numbers appearing in the input. Count the number of integers (without a decimal) separately from the number of floating point numbers (with a decimal, and at least one digit on either side of the decimal).
    What you learn: basic lex syntax, regular expressions in lex, defining symbols for regular expressions, compiling and running
  2. calc.l, calc.y Write the lex file and the yacc grammar for an expression calculator. You need to deal with only '+' and only integers.
    What you learn: yacc syntax, using lex and yacc together
  3. calc.l, calc.y (Overwrite previous files) Enhance the above by adding support for '-' (the binary minus).
    What you learn: left associativity of '-'
  4. calc.l, calc.y (Overwrite previous files) Enhance the above by adding support for enclosing expressions in parentheses '(' and ')'.
    What you learn: your first real context free grammar specification in yacc
  5. calc.l, calc.y (Overwrite previous files) Enhance the above by adding support for '*' and '/'.
    What you learn: operator precedence by order of specification in yacc
  6. text-cleanup.l Implement the text cleanup solution in the tutorial by Victor Eijkhout.
    What you learn: left states and right states in lex parsing
  7. Optional [5HP]: calc.l, calc.y (Overwrite previous files) Add support for simple decimal representation floating point numbers in your calculator.
    What you learn: making yylval a union of possible types.
  8. Optional [5HP]: calc.l, calc.y (Overwrite previous files) Add support for identifier based variables in your calculator, and assignment statements for and using variables.
    What you learn: making yylval a union of possible types, maintaining a symbol table.
  9. Optional [5HP]: ex1.l, ex1.y Exercise-1 from here.
    What you learn: specifying a grammar, parsing a configuration file using lex/yacc
  10. Optional [3HP]: ex2.l, ex2.y Exercise-2 from the same location.
    What you learn: reading someone else's lex/yacc code
  11. Optional [5HP]: ex3.l, ex3.y Exercise-3 from the same location.
    What you learn: integrating parsed configuration parameters into the main program
  12. Optional [10HP]: ex4.l, ex4.y Exercise-4 from the same location.
    What you learn: maintaining a symbol table as you parse an input file
